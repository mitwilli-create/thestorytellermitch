Case study · a Forward Deployed Creative workflow on the ElevenLabs stack

One script in, a produced short out.broll-pipeline

A reusable production workflow for newsroom and brand content teams shipping short-form video on the ElevenLabs stack. Built as a working answer to the Forward Deployed Creative brief: forward-deployed meaning an embedded builder who ships inside the customer's own stack, not a designer for hire. It takes a plain-text script and orchestrates narration, score, sound, and an optional dub into a finished, captioned cut, and it logs every generation step to a costed, reproducible run manifest.

The point is not the 50 seconds. The point is the loop: understand a production pipeline, prototype an AI-native replacement, and package it so a customer can copy it. That loop is the job.

The teardown

The problem, in business terms. A comms or newsroom team needs to ship short explainers on a daily cadence without burning out editors. Today that means a scripter, a voice artist, an editor cutting b-roll (the supporting footage laid over a narration), and a captioning pass, measured in hours per finished minute. The question a forward-deployed creative gets asked is not "can AI make a video," it is "can you turn our existing pipeline into one a small team runs before lunch, and can you prove what it costs."

The shape of the answer. One input file, input/script.md, holds the whole brief: one beat per block, each with a line of voiceover, a visual prompt, and a target duration. The orchestrator parses that into beats and fans them across the ElevenLabs modalities, then ffmpeg cuts everything to the voiceover's timing and burns the caption track. Every stage caches its output keyed by a content hash, so a re-run only regenerates what actually changed. That is the difference between a demo and a workflow a team will keep using.

Model choices, and one honest constraint. The audio stack runs end-to-end through the API: Text-to-Speech for narration, Eleven Music v2 for a licensed, commercially-cleared score, the Sound Effects API for accent beats, and Dubbing for a localized cut. ElevenLabs' 2026 image and video generation lives in ElevenCreative Studio and has no public REST endpoint, so the pipeline treats the visual step as a product-in-the-loop stage: it generates the prompts programmatically, hands them to Studio, and pulls the results back through a capture shim (a small utility that grabs Studio's rendered output back into the pipeline). Building a clean workflow around a real product boundary, rather than routing to a competitor's video API, is deliberate. It stays inside the customer's stack and demonstrates the exact instinct the role calls for.

The manifest is the evidence. Every API call appends to output/run-manifest.json with model, params, cost, and latency. On the run that produced the short above, narration, score, sound effects, and a Spanish dub all executed live for $0.696 total; only the visual stage still routes through ElevenCreative Studio's product-in-the-loop boundary. The manifest is committed in the public repo. That single file answers the customer's real question, "what does one finished minute cost," without a spreadsheet or a guess.

Where a human still touches it. Script approval, brand and legal sign-off, voice selection, and a caption QA pass all stay human. Naming those seams is not a weakness to hide; it is what separates an enterprise-realistic workflow from a toy. The pipeline removes the mechanical hours and leaves the judgment where it belongs.

The pipeline
input/script.md │ ▼ [1] parseScript → beats: [{ vo, visualPrompt, seconds }] │ ├─▶ [2] voiceover (ElevenLabs TTS API) → .cache/vo/*.mp3 ├─▶ [3] visuals (ElevenCreative Studio + shim) → .cache/broll/*.mp4 ├─▶ [4] score (Eleven Music v2) → .cache/music.mp3 └─▶ [5] sfx (Sound Effects API, optional) → .cache/sfx/*.mp3 │ ▼ [6] assemble (ffmpeg: cut b-roll to VO timing, mix stems, burn captions) → output/short.mp4 │ ▼ [7] dub (Dubbing API → target language, optional) → output/short.<lang>.mp4
LIVE = this stage executed against the paid API to produce the cut above. WIRED = the code path is complete and tested; it activates the moment the account's credit plan covers it. "Stems" below = the separate audio layers (voice, music, effects) mixed into the final track.
02
Voiceover live
Each beat's narration is generated through POST /v1/text-to-speech/{voice_id}. A cloned voice makes the whole short sound like one presenter.
03
Visuals Studio-in-the-loop
Prompts are built programmatically per beat and handed to ElevenCreative Studio, then pulled back through the capture shim. No public video endpoint yet, so the workflow wraps the product boundary instead of leaving the stack.
04
Score live
Eleven Music v2, text-to-music or video-to-music against the rough cut. Trained on licensed data and cleared for commercial use, which removes the sync-fee and clearance friction that stops brand teams from shipping.
05
Sound effects live
Optional accent beats generated per cue through the Sound Effects API and mixed under the voiceover.
06
Assemble live
ffmpeg cuts each b-roll clip to its voiceover length, mixes the stems, and burns the caption track from the pipeline's own timing. Deterministic and free.
07
Dub live
One command produces a localized cut through the Dubbing API. The localization proof a media customer asks for first.
