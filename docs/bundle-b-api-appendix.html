One Script, Seven Markets · API Appendix

Built as part of the One Script, Seven Markets spec bundle for the ElevenLabs Forward Deployed Creative role. Endpoints and model IDs below are the ones this bundle was actually produced with (request IDs in the receipts CSV), checked against the live API reference 2026-07-13. Endpoints and model names drift; the reference at elevenlabs.io/docs is the source of truth. MERIDIEM is a fictional canned Spanish white wine created for the demo.

The two Flows templates in this kit (multi-market ad variant, social cutdown) are click-by-click canvas builds. This appendix is the same workflow for an engineer: every node mapped to the API call you would script. It is not hypothetical. The demo set itself was produced through these HTTP endpoints by a scripted pipeline, and every call below has a logged request ID in the receipts. Flows and the API are two front doors to the same machinery; a team can start in the canvas and graduate to a script without changing the workflow.

One market export, in pseudo-code

The shape of the whole workflow, one market at a time. Error handling, caching, and receipt logging trimmed for readability; the real pipeline logs every request ID and cost into the manifests behind the receipts CSV.

const XI = 'https://api.elevenlabs.io';
const H  = { 'xi-api-key': KEY, 'Content-Type': 'application/json' };

// 1. voice: one TTS call per beat, same brand voice_id every market
for (const beat of market.beats) {
  const r = await fetch(`${XI}/v1/text-to-speech/${BRAND_VOICE_ID}`, {
    method: 'POST', headers: H,
    body: JSON.stringify({ text: beat.vo, model_id: 'eleven_v3' }),
  });
  beat.voice = await r.arrayBuffer();          // mp3 bytes
}

// 2. market music bed (the master's visuals stay untouched)
const bed = await fetch(`${XI}/v1/music`, {
  method: 'POST', headers: H,
  body: JSON.stringify({
    prompt: market.musicPrompt,                 // e.g. 'jazz-tinged, brushed drums, aperitif mood'
    music_length_ms: SPOT_MS + 2000,
    model_id: 'music_v2', force_instrumental: true,
  }),
});

// 3. shared ambience (generated once, reused by every market)
const air = await fetch(`${XI}/v1/sound-generation`, {
  method: 'POST', headers: H,
  body: JSON.stringify({ text: 'warm evening rooftop air, distant city', duration_seconds: SPOT_S }),
});

// 4. compose: master visuals + per-beat voice + bed + ambience.
//    Flows does this in the Composition node; a script owns the mix.
//    Here: ffmpeg filtergraph, voice on top, measured linear gain, -14 LUFS.
mix({ visuals: MASTER_MP4, voice: market.beats, music: bed, ambience: air });

// 5. verify captions word-for-word against the finished mix
const stt = await fetch(`${XI}/v1/speech-to-text`, {
  method: 'POST', headers: { 'xi-api-key': KEY },
  body: form({ file: market.mix, model_id: 'scribe_v2' }),
});
diff(stt.words, market.script);                 // any mismatch is a no-ship

Cost instrumentation is part of the workflow

Every call above returns enough to price itself: TTS bills per character, Music per minute, SFX per generation, video per second. The pipeline writes each call's cost and request ID into a manifest as it runs, which is how this kit can publish a CSV of 94 logged calls instead of an estimate. If you script one thing from this page, script that: the receipts are what turn a cost claim into a cost proof.

Model IDs cited from the live models reference (elevenlabs.io/docs, checked 2026-07-13): eleven_v3 (GA), eleven_multilingual_v2 (GA), music_v2, scribe_v2. Treat model choice as a versioned decision: name it in the template, cite the reference, and re-check before a paid run.
