Flows Template A · Article to Audio Edition

Built as part of the Article to Audience spec bundle for the ElevenLabs Forward Deployed Creative role. Cost figures are measured actuals from the bundle's logged run manifest (2026-07-12) or published list rates as cited inline. The Harbor Ledger is a fictional publication created for the demo. No company named in these documents is claimed as an ElevenLabs client.

Back to the review path

What it does: Turns any written article into a branded, narrated audio edition with verified captions, in every language your audience reads you in. Paste text in, get publish-ready audio out, with the same publication voice on every story.

Who gets the most out of it: Digital publishers and newsrooms that want an audio edition without hiring narrators per story, and content teams localizing daily briefs for more than one market.

Build time in the ElevenLabs UI: about 25 minutes the first time, under 5 to reuse once saved as a template.

Inputs and outputs

InArticle text (any length), publication voice ID, target language list
OutNarrated master audio (EN), localized audio per language, caption files verified against the audio, optional music bed under the read

The flow at a glance

[Text: article]──▶[Text to Speech: publication voice]──▶[Composition]──▶ audio edition (EN)
[Text: music prompt]──▶[Music]───────────────────────────▶[Composition]
[Text: article ES]──▶[Text to Speech: same voice, ES]──▶ audio edition (ES)   (UI lane)
        or
narration file ──▶ Dubbing (separate product) ──▶ voice-preserving ES track    (dub lane)

Click-by-click build (ElevenCreative Flows)

Numbered steps assume a signed-in ElevenLabs account with ElevenCreative access. Where a step happens outside Flows, the step says so.

Part 1: the narration spine (about 8 minutes)

  1. From the ElevenCreative dashboard, select + New Flow. You get a blank canvas.
  2. Right-click the canvas (or use the bottom toolbar) and add a Text node. Paste the article body into it. Keep the headline out; you will read that separately if you want a spoken intro.
  3. Add a Text to Speech node the same way.
  4. Drag from the Text node's output port to the Text to Speech node's input port to connect them.
  5. In the Text to Speech node, pick the publication voice. Use one designed or cloned voice for every story so listeners learn the sound of your brand. Set the model to the current multilingual model so the same voice can read other languages later.
  6. Hover over Run on the Text to Speech node to see the credit cost before you spend it, then run it. Listen to the take. If the read is flat, adjust the voice settings and re-run: only this node and nodes downstream of it regenerate.

Part 2: the music bed (about 5 minutes, optional)

  1. Add a second Text node with a short music brief, for example: "warm, understated editorial underscore, felt piano, low dynamic range, sits under a spoken voice, instrumental only."
  2. Add a Music node, connect the text node to it, and run it.
  3. Add a Composition node. Connect both the Text to Speech output and the Music output into it. Set the music track well under the voice; the voice must stay unambiguously on top. Preview inside the Composition node.

Part 3: localization (about 7 minutes)

Two lanes. Pick per story.

  • UI lane (all inside Flows): duplicate the article Text node, replace the body with the translated text, and connect it to a second Text to Speech node using the same publication voice on the multilingual model. Run it. Same voice, new language, fully non-destructive: the English branch is untouched.
  • Dub lane (voice-preserving, outside Flows): download the narration audio and run it through Dubbing in the ElevenLabs dashboard (Products, then Dubbing) with the target language set. Dubbing carries the original voice's delivery into the target language. This is the lane the demo video in this bundle used for Spanish. Dubbing is not a Flows node today; treat it as a one-click side trip and say so to customers rather than papering over it.

Part 4: captions, verified (about 5 minutes, outside Flows)

  1. Run the finished narration through Speech to Text (Scribe) via the dashboard or API to get word-level timestamps.
  2. Diff the transcript against the source article. In this bundle's demo the read came back with 65 of 68 source words matched exactly, and every miss was a formatting difference (the model wrote "400" for "four hundred"), so the read itself was verbatim. That diff is your caption QA receipt: captions built from the STT output are verified against what was actually spoken, not against what you hoped was spoken.
  3. Export the caption file (SRT or VTT) from the word timestamps. Repeat for each language track.

Part 5: save it as a template (about 3 minutes)

  1. In the Flow, mark the article Text node (and the translated text node if you use the UI lane) as input nodes, and mark the Composition output as the output node. At least one output node is required.
  2. Run the template once end to end to validate it.
  3. Share it: Workspace so the whole team can run it, or Link only for a specific customer. Publishing to Explore submits it for public approval.

Scaled with published list rates (TTS $0.10 per 1,000 characters on the multilingual model, Scribe $0.22 per hour, dubbing $0.33 to $0.50 per minute): an 800-word article (about 4,800 characters, roughly 5 minutes of audio) runs about $0.48 to narrate, under $0.02 to caption-verify, and $1.65 to $2.50 per additional language via dubbing. A five-story daily briefing in two languages prices in single-digit dollars per day. Have the customer check current rates at elevenlabs.io/pricing/api before quoting; prices moved down in June 2026 and may move again.

What the AE should watch for (adoption signal)

The template is adopted when the customer's own producers run it on their own stories without you in the room. Ask for the flow's run history on the second call: repeat runs by named editors on paying workspaces are the signal that counts.

Known boundaries (say them out loud)

  • Speech to Text and Dubbing are separate ElevenLabs products today, not Flows nodes. The guide routes around that honestly; do not promise a single-canvas version until it ships.
  • The UI localization lane needs translated text as input; translation itself happens in the customer's existing tooling or the dub lane.
  • Voice consistency across languages is strongest on the dub lane, since dubbing preserves the source delivery.
