What it does: Turns an episode outline into a scripted, voiced, sound-designed podcast episode: intro music, a consistent host voice, scene-level sound design, and a mixed master, all on one Flows canvas. Outline in Monday, episode out Monday afternoon.

Who gets the most out of it: Brand and content-marketing teams that want a recurring branded show without studio bookings, and media companies piloting new show formats before committing production budget.

Build time in the ElevenLabs UI: about 30 minutes the first time, under 10 to reuse once saved as a template.

Inputs and outputs

InEpisode outline (bullet points), host voice ID, show music brief, optional second voice for a two-hander
OutFull episode audio with cold open, music, voiced segments, and sound design layered in one Composition, ready for the customer's podcast host

The flow at a glance

Click-by-click build (ElevenCreative Flows)

Part 1: outline to script (about 8 minutes)

  1. From the ElevenCreative dashboard, select + New Flow.
  2. Open the chat side panel and give the Flows Agent your outline with a brief like: "Draft a 4-minute episode script from this outline. Conversational, one host, short sentences, a cold open, and a sign-off that names the show." The agent can create and connect nodes for you; set its permission mode to Approve each the first time so you see every node it adds and every credit it wants to spend before it spends it.
  3. Review the draft in the Text nodes it creates. Edit by hand; the script is the product, spend your judgment here. If you prefer your own LLM for drafting, paste its output into Text nodes yourself, one node per segment (cold open, body segments, sign-off). One node per segment keeps retakes cheap: a flubbed segment re-runs alone.

Part 2: the host voice (about 6 minutes)

  1. Add a Text to Speech node per script segment and connect each Text node to its own TTS node.
  2. Pick one host voice and use it in every TTS node. Same voice every episode; the voice is the show's brand. A designed voice from the voice library or a consented clone both work.
  3. Hover Run on each TTS node to see the credit cost, then run them. Listen to every take end to end. Re-run only the segments that read flat: the non-destructive canvas means the good takes stay put.

Part 3: music and sound design (about 8 minutes)

  1. Add a Text node with the show's music brief, for example: "confident mid-tempo theme, warm synth and light percussion, builds for 6 seconds then settles into a loopable bed, instrumental." Connect it to a Music node and run it.
  2. For each scene that earns a sound moment (a cold-open sting, a transition), add a Sound Effects node with a specific brief. Name the physical source and the space, for example "single soft page turn in a quiet room, close mic," not "whoosh." Sparse is the craft: effects sit on transitions, ducked under the voice, never mid-sentence attention grabs.
  3. Add a Composition node and connect every TTS output, the Music output, and each Sound Effects output into it. Order the segments on the Composition timeline, tuck the music under the voice, and place effects on the cuts. The voice stays unambiguously on top at every moment; if music ever competes with the host, pull it down, not the voice up.

Part 4: master and ship (about 5 minutes)

  1. Preview the full Composition top to tail once without touching anything. Note timestamps of anything that pokes out.
  2. Export the composition as the episode master. For finer timeline control, captions, or an episode-art video version, export to ElevenCreative Studio and finish there.
  3. Loudness-check the master in the customer's usual QA path before it goes to their podcast host (streaming platforms normalize around -16 to -14 LUFS; the customer's distributor will have a spec).

Part 5: save it as a template (about 3 minutes)

  1. Mark the outline and script Text nodes as input nodes and the Composition as the output node, run it once end to end to validate, then share to the customer's Workspace.
  2. Hand the customer the run history after episode one: the receipt of what each segment cost is the strongest renewal argument you have.

What the AE should watch for (adoption signal)

Adopted means episode three ships without you on the call. The run history shows the customer's own team re-running the template with new outlines; that, plus the template cited in your deal notes, is the metric that counts.

Known boundaries (say them out loud)

  • Script quality is the ceiling. The agent drafts fast, but a brand voice needs a human edit pass; position the template as removing production labor, not editorial judgment.
  • Multi-host banter with overlapping speech is beyond a clean TTS-per-segment build today; a two-hander works as alternating segments.
  • Flows Agent chat bills by token volume on top of generation credits; keep Approve each on until the customer trusts the spend pattern.
