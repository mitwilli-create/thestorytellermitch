Case study · citation-gated verification

Cites its sources, or says nothing.tax-verification-agent

Filing software optimizes for completion, not accuracy. This agent verifies a filing against a four-layer knowledge base: federal code, state statutes, IRS publications, and prior-year precedents, and it refuses to return a verdict without a citation chain. In practice it flagged a roughly $19K multi-state allocation error that a confident piece of commercial software had pre-filled and defended as correct. No citation, no answer. Citation-gating is an old newsroom rule written as code: no source, no story. I enforced it as a journalist and a comms lead; now my agents enforce it too.

What it is

Not a replacement for tax software. A pressure-test for it: a second system that has to show its work, pointed at the output of a first system that never does. Every claim in a verdict traces to a retrievable source, and "unknown" produces a dispute, not a guess.

Optimizes for getting you to the finish line. When questioned, it defends its own math with confidence and no sources. You are trusting a black box with a material number.
tax-verification-agent

What it does

The catch that validates it. A live filing claimed a state income exclusion. The software said the calculation was correct. The agent pulled the statute. It found the exclusion applies only when a specific condition is met. That condition was absent from the filing. So it returned a dispute, citation attached. The discrepancy was material, about $19K. That is the gap this is built for: not confidence versus confidence, but confidence versus evidence.

Three components, one constraint. A retriever queries the four-layer knowledge base with ranked retrieval. A verifier runs each parsed claim through a Claude prompt chain: claim, then citation, then verdict. A reporter assembles a structured output with source attribution on every line. The constraint that binds them: any assertion lacking a grounded citation is blocked before it reaches the report.

The refusal is the feature. A confident wrong answer about taxes is worse than no answer. So the gate is not a stylistic preference, it is the architecture: citation-gated retrieval forces the model to ground every output in a retrievable source, and the system's credibility comes from what it refuses to say without evidence. It is the same constraint layer as the executive voice pipeline I built at Google, applied to a domain where being wrong has a dollar figure.

Built like it's going to production. Structured JSON input with a published schema, a layered knowledge base that separates federal code from state guidance from IRS publications from precedent notes, and reports designed to be read by a person deciding whether to amend a filing. A personal project, held to a production bar.

How it works
filing input (JSON, published schema) │ ▼ [1] parse → discrete claims │ ▼ [2] retrieve (four-layer KB, ranked) federal code · state statutes · IRS pubs · precedents │ ▼ [3] verify (Claude) → claim → citation → verdict │ ▼ [4] gate → citation chain present? yes: verdict · no: blocked │ ▼ [5] report (structured, every line source-attributed)
01
Parse the filing into claims
The filing input arrives as structured JSON and is decomposed into discrete, checkable claims: an exclusion applied, an allocation made, a deduction taken. Each claim is verified on its own, so one confident error cannot hide inside an overall "looks right."
02
Retrieve across four layers
Ranked retrieval queries federal code, state statutes and DOR guidance, IRS publications, and prior-year precedent notes as separate layers. The layering matters: a state exclusion question should be answered by the state statute, not by whichever passage happens to embed nearest.
03
Verify with a chained prompt
Claude evaluates each claim against the retrieved passages in a fixed chain: state the claim, produce the citation, then and only then render the verdict. The order is the discipline; the verdict cannot arrive before the evidence.
04
Block what can't cite
Any assertion without a grounded citation is blocked. Disputes return with the knowledge-base match and the failing condition spelled out, so the output is actionable: not "this looks wrong" but "this exclusion requires condition Y, and condition Y is not present."
05
Report with attribution
The reporter produces a structured markdown report: claim, status, knowledge-base match, verdict. Every line carries its source, so a person reviewing the report can check the agent the same way the agent checked the software.
Why it matters

Citation-gated retrieval is not a prompt trick, it is a constraint layer, and it is the difference between an AI system you can check and one you have to believe. The validation here wasn't a benchmark: it was a live filing, a confident piece of commercial software on the other side, and a structured agent that disagreed with a citation. In a domain where the cost of a wrong answer is denominated in dollars, the system that must show its work wins.

I didn't trust the software's confidence. I trusted the citation. That is the whole design, in one sentence.
